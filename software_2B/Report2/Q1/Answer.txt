【エラーの原因】
同じメモリ領域を2回削除しようとしたため発生した。


【発生箇所】
main.cpp
    Message obj1("Hello World.");
    Message obj2 = obj1;

出力↓
obj1.message のアドレス: 0x5a38a545eeb0
obj2.message のアドレス: 0x5a38a545eeb0


1. Message obj2 = obj1; でコピーが発生
2. コピーコンストラクタが実装されていないため、デフォルトコピーが実行される
3. デフォルトコピーはポインタのアドレス値をそのままコピーする
4. 結果：obj1.messageとobj2.messageが同じメモリ領域を指す
5. プログラム終了時：
    ・obj2のデストラクタでメモリ削除
    ・obj1のデストラクタで同じメモリを再度削除 → エラー


一言でまとめると
ポインタを持つクラスで適切なコピーコンストラクタがないため、2つのオブジェクトが同じメモリを共有し、
それぞれが同じメモリを削除しようとしてエラーになった。


【根本原因】
ポインタメンバ変数を持つクラスで、適切なコピーコンストラクタ（深いコピーを行う）が実装されていないため、
複数のオブジェクトが同一のメモリ領域を共有し、重複してメモリ解放を行ってしまうことが原因です。
